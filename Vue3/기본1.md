# 하드웨어와 소프트웨어

- 시스템 리소스(system resource)

  > 정의: 컴퓨터의 가상 구성 요소를 의미한다.
  >
  > 예: 파일 핸들, 네트워크 소켓, 메모리 영역

- 컴퓨터 하드웨어(computer hardware, H/W)

  > 정의: 컴퓨터의 물리적 부품을 의미한다.
  >
  > 예: 케이스, 중앙 처리 장치(CPU), 모니터, 자판, 컴퓨터 기억 장치, 그래픽 카드, 사운드 카드, 메인 보

- 컴퓨터 소프트웨어 = 소프트웨어(computer software = software)

  > 정의: 컴퓨터 시스템에서 실행되는 프로그램과 그 프로그램이 작동하는 데 필요한 모든 관련 파일과 데이터(모든 소프트웨어를 총칭한다.)
  >
  > 구성 요소: 프로그램, 라이브러리, 구성 파일, 데이터 파일, 리소스 파일, 문서

  - 프로그램 소프트웨어 = 컴퓨터 프로그램(program software = computer program)

    > 정의: 컴퓨터에서 실행될 때 특정 작업을 수행하는 명령어 집합이다.(사용자에게 직접 노출되지 않을 수 있다.)
    >
    > 예: 파일 시스템 관리 프로그램

    - 응용 소프트웨어 = 응용 프로그램 = 애플리케이션(application software = application)

      > 정의: 운영 체제에서 실행되는 모든 소프트웨어이다.(사용자에게 직접 노출된다.)

      - 오피스 제품군: 워드 프로세서, 스프레드시트, ...
      - 컴퓨터 통신: 웹 브라우저
      - 멀티미디어: 미디어 재생기, 그래픽 프로그램
      - 분석 소프트웨어: 컴퓨터 대수학 시스템, MathCAD, ...
      - 협업 소프트웨어: 오픈 소스, 블로그, ...
      - 사무용 소프트웨어: 백 오피스, 고객 관리(CRM), ...
      - 데이터베이스: DBMS
      - 교육용 소프트웨어

  - 시스템 소프트웨어(system software)

    > 정의: 응용 소프트웨어를 실행하기 위한 플랫폼을 제공하고 컴퓨터 하드웨어를 동작, 접근할 수 있도록 설계된 컴퓨터 소프트웨어이다.

    - 부트 로더
    - 운영 체제
    - 장치 드라이버
    - 셸
    - 로더
    - 라이브러리: C 표준 라이브러리, ...
    - 부호 생성 도구: 컴파일러

# 운영 체제

- 운영 체제(Operating System, OS)

  > 정의: 사용자의 컴퓨터 하드웨어, 시스템 리소스를 제어하고 프로그램에 대한 일반적 서비스를 지원하는 시스템 소프트웨어
  >
  > 

- API

  > 정의: 응용 프로그램 간의 상호작용 및 연결
  >
  > 넓은 정의: 하드웨어, 소프트웨어, 네트워크 등의 시스템 구성 요소 간의 상호작용을 관리하고 제어하는 인터페이스
  >
  > 특징: 인터페이스 제공(기능 사용), 추상화(세부 사항 상관 없이 간단하게 사용), 표준화(일관된 방식으로 기능 제공)

  - 하드웨어 제어

    - 드라이버 API

      > 정의: 하드웨어 장치와 운영 체제 간의 통신을 관리
      >
      > 예: 그래픽 카드 드라이버 API

  - 소프트웨어 간 상호작용

    - OS API

      > 정의: 운영 체제의 기능을 이용하기 위한 인터페이스를 제공
      >
      > 예: 파일 시스템 API

    - 웹 API

      > 정의: 웹 서버와 클라이언트 간의 통신을 관리
      >
      > 예: REST API

  - 네트워크 제어

    - 네트워크 API

      > 정의: 네트워크 상의 데이터 전송 및 통신을 관리
      >
      > 예: 소켓 API

  - 인터페이스 제어

    - DOM API

      > 정의: 웹 페이지의 구조와 내용을 동적으로 변경할 수 있는 인터페이스를 제공
      >
      > 예: document.getElementById 함수

- 메타데이터(metadata = metainformation)

  > 정의: 데이터에 대한 데이터
  >
  > or 어떤 목적을 가지고 만들어진 데이터
  >
  > 예: 도서관에서 사용하는 서지기술용으로 만든 것

- 태그(tag)

  > 정의: 어떤 정보에 메타데이터로 부여된 키워드 또는 분류
  >
  > or 문서의 구조와 내용을 정의하는 데 사용되는 요소
  >
  > 예: \<html\>, \<head\>, \<title\>, \<body\>, \<p\>, \<a\>, \<img\>, \<li\>, \<div\>, \<span\>, ...

  - 속성(property)

    > 정의: 태그에 추가 정보를 제공
    >
    > 예: id, class, src, alt, style, ...
